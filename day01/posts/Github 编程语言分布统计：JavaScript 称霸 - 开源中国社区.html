Github 编程语言分布统计：JavaScript 称霸

陨落人间 发布于： 2014年11月17日 (33评)
分享到： 
收藏 +41

3月19日，深圳源创会火热报名中， React Native 开发实践 »  

英文来自：Asas Dhamani

【编者按】Github作为全球最大的开源项目托管平台，每天都会有许多项目托管在上面，有些程序员会好奇，这么多项目，有哪些项目是基于Java或者Python开发的呢？本文作者或许就是带着这样的好奇，对Github上各门编程语言进行了统计。但他并未透露具体的统计方法，本文数据仅供大家参考。

以下为译文

我一直想了解一下各个语言在Github上的分布情况，虽然以前有过相关文章出现，但我觉得他们写的都不是很清楚，为了更加地一目了然，我决定自己动手。

Stars

目前，Github上有68个存储库（repositories，下文简称仓库）的stars是超过10000个的。

在这些仓库中，基于JavaScript开发的仓库占到了55.7%，其次分别是CSS、Ruby。在Github上，前端项目获得了较高的关注度。

下面再来看看Stars前10名的仓库情况：

不负众望，JavaScript占到了70%，也就是说，在排名前十个仓库中，有7个都是基于JavaScript开发的。

目前，Stars大于1的仓库有2209178个。


在这次统计中，JavaScript以23.2%的份额又排在了第一位。紧随其后的分别是Ruby和Java。

Forks

Github上有10个仓库的forks数超过了8000，下面来看看这些仓库的语言分布情况。

在这次统计中，CSS跃居到了第一位。而JavaScript和Ruby并列在了第二位。

而在整个Github上，forks数大于1的仓库有944872个。

不出所料，JavaScript在所有的forks仓库中排名第一，其后分别是Java、Python。

Repositories

根据Github上的统计，其上面的整个仓库有16.9M（16900000）个。

下面是这些仓库所使用的编程语言中，排名前十位的分别是：

在所有的仓库中，其中有21.2%个仓库是基于这十门语言开发的。而JavaScript又是排名第一。紧随其后的则是Java和Python。

Issues

Github官方统计有42.2M个问题，然而，我只统计到了12.4M个。

下图数据来自于我所统计的Issues。

Closed

有开就有关，下图是对所有关闭仓库的统计。

图片说明一切，JavaScript又占据了第一。
